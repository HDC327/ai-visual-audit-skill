# Skill：视觉审核决策表（持续更新）

版本：1.3 | 兼容 `skill-review-flow.md` ≥ v2.2 | 被 `skill-review-flow.md` 依赖 | 引用 `skill-review-redlines.md` ≥ v1.0

> 本文件是给 Agent 看的**判断口径与经验背景**，不是机械执行清单。它提供：①节点侦测规则 ②常规决策表（物料 × 场景）③已固化的节点决策表（端午、双11…）④维度定义、维度优先级与风险等级口径 ⑤物料标准下钻路由。
>
> 用法：先按物料 × 场景查常规表确定必审维度，命中节点再叠加节点表附加维度，任务要求/文案永远是最高优先级维度。
>
> 这是一份**活文档**：随真实误判/漏判反馈持续校准。

## 0. 第一原则：角色推理优先，标准是燃料不是枷锁

> 这一节统领全文，优先级高于下面任何一张表。

- **你的身份是资深视觉审核专家，不是规则解析器。** 用判断力看图，模拟「有经验的人会怎么看」，而不是把下面的表当 checklist 逐条勾选。
- **本文件提供的所有标准、节点维度、物料路由、合规红线，都是帮助你推理的「背景知识与证据来源」，不是限制你思考的边界。** 它们的作用是让你的判断**有依据**（填进 `evidence`），不是替你下结论。
- **两条反限制原则**（避免背离注意力导航初衷）：
  - **标准没写到的 ≠ 通过**：没有对应条目时，仍用专家判断力看，不要因为「规范没提」就放过明显问题。
  - **偏离常规模板的 ≠ 必须先改**：突破性、非常规但成立的创意，不因「不像常规好海报」而压低；创意维度按「是否与近期同类高度雷同」判断，而非「是否符合模板」。
- **唯一接近确定性核对的是合规红线**（`skill-review-redlines.md`）：合规是是非题，其余维度都是判断题。即便红线命中，产出仍是「提示用户去看」，最终是否修改或使用由人决定。
- **宁缺毋滥**：表里维度多，不代表每条都要报。只导航到**真正值得用户多看**的点（最多 3 条）。

> 关联文件：合规红线索引 `skill-review-redlines.md`；标准源 `references/content-safety-standards/`、`references/content-quality-standards/`、`references/visual-design-audit-dimensions/`（均**按需懒加载**，命中才 Read，平时不整篇读入）。

## 1. 节点侦测规则

扫描 `title`、`detailRequirementFullText`、`taskCopywriting`，命中任一信号即视为该节点。
信号包含但不限于：节日名、活动名、专属利益话术、专属视觉元素。

| 节点类别 | 触发信号（关键词 / 语义） | 归类到 |
|---|---|---|
| 传统节日 | 端午、粽子、龙舟；中秋、月饼；春节、年货、新春；元宵；清明 | §3 对应节日节点 |
| 大促 | 双11、双十一、618、99 大促、年中大促、品类日、超级品牌日、预售/定金/尾款 | §3 大促节点 |
| 新品 | 新品首发、上新、首发、新款发布、newin | §3 新品节点 |
| 营销日 | 情人节、520、女神节/38、母亲节、父亲节、儿童节、教师节、国庆 | §3 通用节日节点（无专章则套通用） |
| 品牌 | 品牌日、品牌升级、形象片、tvc、视觉规范、vi | §3 品牌节点 |

判定原则：

- **就高不就低**：同时命中大促和节日（如「双11 龙舟季」）时，两套附加维度都叠加。
- **语义优先于字面**：出现「定金 / 尾款 / 预售」即按大促处理，即使没写「双11」。
- 命中但无对应专章 → `detected_node` 填原始关键词，仅叠加 §3「通用节点附加维度」。

## 2. 常规决策表（物料 × 场景）

按物料类型与投放场景确定**必审维度**与**重点**。场景无法判断时用「默认」行。

### 2.1 物料类型 → 基础必审维度

| 物料类型 | 必审维度（基础） | 易错点 |
|---|---|---|
| 海报 / 主视觉 | 主体清晰度、信息层级、文案一致性、品牌/合规要素 | 主体与背景对比不足、文案与画面不符 |
| Banner / 通栏 | 信息层级、利益点可读性、留白与安全区、文案一致性 | 小尺寸下文字糊、利益点不突出 |
| 信息流图 | 第一眼吸引力、主体识别、文案精炼、合规 | 元素过满、卖点不聚焦 |
| 详情页 / 长图 | 模块逻辑顺序、信息完整性、视觉节奏、文案一致性 | 模块跳跃、上下文断裂 |
| UGC / 实拍 | 真实性、清晰度、合规（人物/版权/水印）、与任务匹配 | 盗图、水印、与任务要求跑题 |

### 2.2 投放场景 → 审核重点

| 投放场景 | 审核重点 | 倾向口径 |
|---|---|---|
| App 首焦 | 利益点是否够突出、转化引导是否清晰、移动端小屏可读性 | 利益点弱/引导缺失 → 偏 high |
| Feed 流 | 第一眼吸引力、信息是否过载、与周边内容差异度 | 同质化严重 → 偏 mid |
| 详情页 | 信息完整与逻辑顺序、与上游主图一致性 | 信息断裂 → 偏 mid～high |
| 默认（场景未知） | 主体清晰度 + 文案一致性 + 合规三项必审 | 其余按物料基础维度 |

### 2.3 维度交叉提示

- **文案一致性**为所有物料的必审维度（图片与 `taskCopywriting` / 任务要求是否相符）。
- **合规**为辅但不可省：人物肖像、版权素材、水印、违禁词、绝对化用语。疑似命中时按 `skill-review-redlines.md` 扫描确认（命中即 high）。
- **创意维度说明与局限**：
  - 判断标准为「是否落入安全平庸模板 / 是否与同类高度雷同」，**不**因「偏离常规模板」就判低分，避免压制突破性创意。
  - Agent 在单次 session 内通常只能看当前任务图，无法真正访问「近期同类」历史物料；因此「雷同」判断主要来自 Agent 对行业通用模板的认知，不是精确历史对比。
  - 创意维度置信度通常不应高于「中」，仅供用户参考，不建议单独作为“必须先改”的主要依据。

## 3. 已固化的节点决策表

> 命中节点时，在常规维度之上**叠加**下表附加维度。节点维度通常优先级高于常规维度。

### 3.0 通用节点附加维度（命中任何营销节点都适用）

| 附加维度 | 看什么 |
|---|---|
| 节点氛围契合度 | 视觉调性、配色、元素是否贴合该节点心智 |
| 利益点/主题准确性 | 节点专属话术、日期、玩法是否准确无误 |
| 时效正确性 | 活动时间、场次、预售/正式期标注是否与任务要求一致 |

### 3.1 端午节点

| 附加维度 | 看什么 | 红线 |
|---|---|---|
| 节日氛围 | 粽子、龙舟、艾草、青绿/朱砂等元素是否到位且不堆砌 | 元素缺失显得不像端午；元素过满抢主体 |
| 文化准确性 | 文案祝福语得体（「端午安康」优先于「端午快乐」语境） | 习俗/寓意用错 |
| 利益点 | 礼盒/限定/满减等卖点是否清晰 | 节日氛围盖过利益点 |

### 3.2 双11 / 大促节点

| 附加维度 | 看什么 | 红线 |
|---|---|---|
| 利益点突出度 | 主利益点（折扣/到手价/满减）是否是画面第一视觉重心 | 利益点字号偏小、被装饰淹没 |
| 价格信息准确 | 价格、定金/尾款、场次时间与任务要求一致 | 价格错、预售/正式期标注混乱 |
| 转化引导 | CTA（立即抢购/领券）是否明确可点感 | 无明确引导 |
| 大促氛围 | 红/金/促销视觉强度足够但不廉价 | 氛围弱显得不像大促 |
| 合规 | 价格类绝对化用语、虚假优惠表述 | 「全网最低」等违禁表述 → high |

### 3.3 新品节点

| 附加维度 | 看什么 | 红线 |
|---|---|---|
| 产品表现力 | 产品主图清晰、卖点可视化、质感到位 | 产品糊/小/被遮挡 |
| 差异化记忆点 | 是否有区别于旧款的记忆点，而非套旧模板 | 与历史款高度雷同 |
| 信息准确 | 新品名称、规格、首发信息准确 | 名称/规格错 → high |

### 3.4 品牌节点

| 附加维度 | 看什么 | 红线 |
|---|---|---|
| 品牌气质一致 | 调性、字体、配色与品牌规范一致 | 偏离 VI |
| 视觉克制度 | 高端/简约场景下信息是否克制、留白是否充分 | 元素堆砌、促销感过重 |
| 要素规范 | logo 安全区、最小尺寸、清晰度 | logo 变形/过小/低清 |

## 4. 风险等级判定口径

| 等级 | 判定 | 典型 |
|---|---|---|
| high | 直接影响投放效果或用户核心识别，需修改才能用；或触碰合规红线 | 利益点不可读、价格/规格错误、违禁词、盗图水印、文案与画面严重不符 |
| mid | 有明显改善空间，不强制阻断但建议修改 | 层级偏弱、对比不足、氛围欠缺、同质化 |
| 轻微（不进 risk_items） | 主观偏好或可忽略的小瑕疵 | 个别留白略紧、轻微构图偏好 |

口径提示：

- 拿不准 high/mid 时，看是否**影响用户核心识别或转化**：影响则 high，仅观感则 mid。
- 合规红线一律 high（疑似命中 `skill-review-redlines.md` 即按 high，拿不准时用“需要你再确认”交还用户）。
- 同一问题不要拆成多条；最多保留 3 条最值得看的。
- 发现超过 3 个真实问题时：默认只展开最重要的 3 条；如确需告知，可简单说明“还有其他小问题未展开”。

## 5. 维度类型定义（`dimension_type`）与优先级

### 5.1 四级优先级（问题严重性排序）

> 源自 `visual-design-audit-dimensions/视觉设计稿审核.md`，是官方审核本体。当一单同时有多类问题时，按此顺序决定先暴露哪个、以及风险等级倾向。

**安全/版权 ＞ 规范 ＞ 需求 ＞ 质量**

| 层级 | 含义 | 对应 `dimension_type` | 典型口径 |
|---|---|---|---|
| 安全/版权 | 内容安全红线、肖像/版权/抄袭 | 合规 | 命中 `skill-review-redlines.md` → 一律 high |
| 规范 | 尺寸/分辨率/安全区/页数/最小字号/LOGO 与二维码规范等 | 质量（规范向） | 影响投放或识别 → high，仅观感 → mid |
| 需求 | 是否符合任务要求 / brief / 文案 | 文案一致性 | 与要求严重不符 → high |
| 质量 | 排版/文本/元素/配图/色彩/布光的完成度 | 质量、创意 | 多为 mid，硬伤才 high |

### 5.2 `dimension_type` 取值范围

| 类型 | 范围 | 质量子维度（来自审核本体，供 `dimension` 命名参考） |
|---|---|---|
| 质量 | 清晰度、对比度、构图、层级、留白、可读性、画面完成度、规范符合度 | **排版**（亲密性/对比/对齐/重复/品质感）、**文本**（辨识度/一致性）、**元素**（一致性/合理性/品质感）、**配图**（一致性/合理性/品质感）、**色彩**（合理性/品质感）、**布光透视**（一致性/合理性） |
| 创意 | 吸引力、记忆点、差异度、与同类的雷同程度（见 §2.3 局限说明） | — |
| 合规 | 肖像/版权/水印、违禁元素/违禁词、绝对化用语、时效与价格表述合规 | 详见 `skill-review-redlines.md` |
| 文案一致性 | 图片与任务文案/任务要求是否相符、信息是否准确无遗漏 | — |

> 这六个质量子维度是**命名与推理参考**，帮助你把「哪里不对」说得更具体（如 `dimension: "排版·层级"`），**不是要逐一打分**。仍遵守 §0：用判断力挑出值得看的，不逐条核对。

## 6. clear_dimensions 使用规范

`clear_dimensions` 是向用户传递「这些点可以快速略过」的信号，减少重复核查。

填写规则：

- 只填 Agent **已经主动核查过并认为无明显问题**的维度，不填「没有检查过」的维度。
- 最多列 4 个；超出时按重要性截断。
- 不要填入 `risk_items` 已标注风险的同一维度。
- 示例值：`["主体清晰度", "品牌合规", "文案一致性"]`

## 7. 反馈校准入口

本文件可以根据 `skill-review-optimizer.md` 的静默反馈结果自动校准，但只改稳定、重复出现的问题：

- `false_positive` 集中出现：优先放宽对应物料 / 场景下的判断边界，或把轻微问题降为不展开。
- `missed_issue` 集中出现：补充对应物料 / 节点的易错点，或收紧 high / mid 边界。
- `needs_explanation` 集中出现：增强该维度的依据表达，不一定改变判断标准。
- `execution_violation` 不改 criteria，回到 `SKILL.md` 或 `skill-review-flow.md` 增强输出前自检。
- 合规、安全、版权相关反馈只允许自动收紧或增强提示，不因少量用户反馈自动放宽红线。

自动优化后，应把已消费反馈汇总到 `references/feedback/change-log.md`，并清空 `references/feedback/feedback-log.md` 中对应明细，避免反馈记录无限膨胀。
