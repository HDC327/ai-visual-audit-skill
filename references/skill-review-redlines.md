# Skill：合规红线索引（视觉 / 文案）

版本：1.0 | 被 `skill-review-criteria.md` ≥ v1.3 引用 | 详情源：`references/content-safety-standards/`

> 本文件是给 Agent 的**红线扫描索引**，不是完整规范。它把两份安全文档蒸馏成「类别 → 触发信号 → 红线结论 → 例外 → 详情锚点」，让 Agent **每单快速扫一遍**判断是否疑似踩线，**只有命中候选、且需要确认例外细节时**，才去 Read 对应详情全文，避免整篇读入。
>
> **与决策表的关系**：决策表（`skill-review-criteria.md`）是**判断透镜**，靠角色推理；本红线索引是其中**唯一接近确定性核对**的一层——合规是是非题，不是判断题。但它服务的仍是「注意力导航」：命中即提示用户「这里可能踩线、去看 X 规范」，**最终是否修改或使用仍由人决定**，Agent 不做终审、不自动封禁。
>
> **口径**：疑似命中红线 → 内部风险等级一律 **high**，默认展示为「必须先改」或「需要你再确认」。**拿不准是否构成红线 → 明确交还用户确认**，宁可提示也不漏过合规风险。

---

## 0. 使用方式（Agent 内部推理）

1. 扫 `taskCopywriting`、`detailRequirementFullText`、`title` 与画面元素，对照下表「触发信号」。
2. 命中信号 → 先看「例外（✅）」是否适用：
   - 明显落在例外内 → 不报，或仅在 `confidence_reason` 一句话带过。
   - 不在例外内、或不确定 → 报 high 风险，需要精确表述/例外边界时再 Read 详情全文。
3. **没命中任何信号 → 不要为了凑数硬报合规**；合规红线只在确有疑似信号时触发。
4. 红线判断**优先级最高**（见 `skill-review-criteria.md` §5 四级：安全/版权 ＞ 规范 ＞ 需求 ＞ 质量）。同一单既有红线又有质量问题时，红线先行。

> 详情文件：视觉类 → `references/content-safety-standards/视觉禁用元素.md`；文案类 → `references/content-safety-standards/文案禁用词汇.md`。下表「详情锚点」给出文档内的小节定位。

| 触发信号 | 红线结论 | 备注 |
|---|---|---|

## 4. 维护说明

- 本索引随 `content-safety-standards/` 源文档更新而校准；源文档结构变动时同步「详情锚点」。
- **只放可扫描的触发信号与结论**，完整图示/逐条表述留在源文档，避免索引膨胀回退成「整篇规范」。
- 新增红线类别时，先确认它确属**是非题**（确定性核对）；属于判断题的（如氛围、层级、创意）应回到 `skill-review-criteria.md` 决策表，不进本索引。
